# Observations schema

Full reference for the `observations` resource in EA-DX Package v0.1.0.

The JSON Schema source is at [`standards/eadx/schemas/observations.json`](https://github.com/craigm26/energy-ai-education/blob/main/standards/eadx/schemas/observations.json).

## Field reference

### `ts` — timestamp

**Type:** string (ISO 8601 datetime)  
**Required:** yes

Timestamp of the observation. **Must include a UTC offset** (e.g. `2024-06-15T14:00:00-07:00` or `2024-06-15T21:00:00Z`). Naive timestamps (without offset) are not valid.

!!! warning "Interval convention"
    In IEC CIM and Green Button, timestamps typically mark the **end** of the interval. Check your source data's convention and document it in the dataset card.

### `tz` — IANA timezone

**Type:** string (IANA tz database)  
**Required:** no (but recommended when `ts` uses a fixed numeric offset)

Example: `America/Los_Angeles`, `Europe/Berlin`, `UTC`.

Allows correct handling of DST transitions when resampling across clock changes.

### `entity_type` — type of entity

**Type:** enum  
**Required:** yes

| Value | Description |
|-------|-------------|
| `meter` | Revenue or interval meter (residential, commercial, industrial) |
| `inverter` | Solar or battery inverter |
| `feeder` | Distribution feeder |
| `plant` | Generating plant or aggregate generation asset |
| `market_node` | Pricing node, hub, or zone in a wholesale market |
| `building` | Building-level aggregation (energy consumer) |
| `ev_charger` | EV charging station |
| `battery` | Battery energy storage system |
| `other` | Use sparingly; document in dataset card |

### `entity_id` — opaque entity identifier

**Type:** string  
**Required:** yes

**Must be stable and opaque.** Must not be a real customer account number, utility account ID, or any other PII-linked identifier.

Use synthetic IDs (`MTR-0001`, `FDR-WEST-A`, `HUB-CAISO-NP15`) or hashed/pseudonymised identifiers.

### `measurement` — measured quantity

**Type:** enum  
**Required:** yes

| Value | Description |
|-------|-------------|
| `active_power` | Real power (W, kW, MW, GW) |
| `reactive_power` | Reactive power (VAR, kVAR, MVAR) |
| `apparent_power` | Apparent power (VA, kVA, MVA) |
| `voltage` | Voltage magnitude (V, kV) |
| `current` | Current magnitude (A, kA) |
| `frequency` | System frequency (Hz) |
| `power_factor` | Dimensionless or `pu` |
| `energy` | Accumulated energy (Wh, kWh, MWh, GWh) |
| `temperature` | Ambient or equipment temperature (Cel) |
| `price` | Electricity price (USD/MWh, EUR/MWh, etc.) |
| `price_forecast` | Model-generated price forecast |
| `soc` | State of charge — batteries/EVs (pct) |
| `irradiance` | Solar irradiance (W/m²) |
| `wind_speed` | Wind speed (m/s) |
| `other` | Document in dataset card |

### `value` — numeric value

**Type:** float  
**Required:** yes

The numeric measurement value. Use the `unit` field to specify units. NaN or missing values should be represented as empty cells, not as sentinel values (−9999, −999, etc.).

### `unit` — unit of measurement

**Type:** enum (UCUM-aligned)  
**Required:** yes

See the [JSON schema](https://github.com/craigm26/energy-ai-education/blob/main/standards/eadx/schemas/observations.json) for the full allowed set.

### `quality_flag` — data quality / provenance

**Type:** enum  
**Required:** yes

| Value | Meaning |
|-------|---------|
| `measured` | Direct sensor reading, no post-processing |
| `estimated` | Estimated from meter read or statistical method |
| `imputed` | Missing value filled by interpolation or model |
| `simulated` | Generated by a simulation or synthetic data process |
| `redacted` | Value suppressed for privacy/sensitivity; structure preserved |
| `unknown` | Provenance not available |

### `source` — origin of data

**Type:** enum  
**Required:** yes

| Value | Meaning |
|-------|---------|
| `sensor` | Physical sensor or meter |
| `sim` | Simulation or synthetic generator |
| `market` | Market clearing or price publication |
| `derived` | Computed from other measurements |
| `manual` | Manually entered |
| `other` | Document in dataset card |
