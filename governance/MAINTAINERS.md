## Contributor ladder

### Contributor
- Any merged PR
- Can open issues and participate in discussions

### Reviewer
- 3+ merged PRs in a specific domain
- Invited by an existing maintainer
- Auto-assigned as reviewer for their domain (CODEOWNERS)

### Maintainer
- Consistent high-quality contributions over time
- Demonstrates community trust and judgment
- Nominated and approved by existing maintainers
- Can merge PRs and manage releases

## Decision-making

- **Routine changes** (docs, bug fixes, small additions): any maintainer can approve and merge
- **Schema changes** (EA-DX spec): requires discussion in an issue + at least one maintainer approval
- **Governance changes** (this file, CODE_OF_CONDUCT, SECURITY): requires all active maintainers to agree
- **Breaking changes**: open a Discussion, allow 7 days for community input

## Stepping down

Maintainers who are stepping back should open a PR updating this file and transfer active issues to remaining maintainers.
